**Временная сложность**: $O(logn)$

**Пространственная сложность**: $O(1)$

Описание

Формально это бинарный поиск, но с небольшими изменениями. Мы заводим отрезок, который изначально охватывает весь массив. Далее смотрим на центральный элемент массива и сравниваем его с соседними элементами. Если мы нашли точку поворота, то возвращаем соответствующий элемент. Если точку поворота не нашли, уменьшаем отрезок для поиска.

Алгоритм:
- Заводим два указателя: $left$ и $right$
- $left$ указывает на начало массива, $right$ указывает на конец массива
- Считаем значение центрального элемента между $left$ и $right$ ($mid$)
  - Если следующий элемент меньше $mid$, возвращаем значение следующего элемента
  - Если предыдущий элемент больше $mid$, возвращаем $mid$
  - Если $mid$ больше первого элемента, то поворот массива где-то дальше, поэтому увеличиваем $left$ на 1
  - Если $mid$ меньше первого элемента, то поворот массива где-то раньше, поэтому уменьшаем $right$ на 1
