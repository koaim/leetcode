**Временная сложность**: $O(a+b)$, где $a$ - это длинна `headA`, $b$ - это длина `headB`

**Пространственная сложность**: $O(1)$

Алгоритм:
- Проходим по первому односвязному списку и считаем его длину `lenA`
- Проходим по второму односвязному списку и считаем его длину `lenB`
- Если последние элементы каждого из списка не равны, возвращаем `nil`, так как это означает, что списки не пересекаются
- Снова заводим два указателя `currA` и `currB`, которые указывают на головы списков
- Сравниваем длины списков
  - Если первый список длиннее, то двигаем укзаатель `currA` на `lenA - lenB`
  - Если второй список длиннее, то двигаем указатель `currB` на `lenB - lenA`
- Последовательно двигаем каждый из указателей на один элемент вперёд, пока они не пересекутся в одной точке
- Возвращаем `currA` или `currB`
